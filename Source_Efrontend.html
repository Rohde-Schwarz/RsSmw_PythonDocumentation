get_cmode() → RsSmw.enums.All[source]
# SCPI: [SOURce<HW>]:EFRontend:CMODe
value: enums.All = driver.source.efrontend.get_cmode()

Sets the mode of the SSL control connection between R&S SMW and external frontend.

return

fe_conn_mode: AUTO| LOCK| RXTX AUTO The R&S SMW locks external frontend, when activating the RF output at the R&S SMW (‘RF’ ‘On’) for output of the IF signal. The R&S SMW unlocks external frontend, when deactivating the RF output at the R&S SMW (‘RF’ ‘Off’) . LOCK The external frontend is locked permanently. No other instrument can take over control. RXTX Requires an R&S FE50DTR connected to the R&S SMW. Connection mode for simultaneous Rx (receive) operation and Tx (transmit) operation of an R&S FE50DTR. Set this mode, if your test setup requires a release of connection lock. A connected vector signal analyzer can lock the external frontend for Rx operation.
